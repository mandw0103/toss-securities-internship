# 커밋 스킬

## 트리거

민서가 커밋을 요청했을 때 사용한다.

## 커밋 메시지 규칙

- **gitmoji + 한글**로 작성한다
- 무엇을 했는지 구체적으로 작성한다
- 예시:
  - `✨ 공통 헤더 네비게이션 및 페이지 라우팅 구현`
  - `🐛 홈 화면 초기 로딩 시 깜빡임 현상 수정`
  - `📝 홈 화면 기획서 작성`

## 커밋 단위

사이클의 각 단계가 커밋 단위이다:

| 단계 | gitmoji | 예시 |
|------|---------|------|
| 기획서 확정 | 📝 | `📝 홈 화면 기획서 작성` |
| FE 스펙 확정 | 📝 | `📝 홈 화면 FE 스펙 작성` |
| ADR 작성 | 📝 | `📝 ADR-001 PR 본문 자동화 결정 기록` |
| 구현 | ✨ / 🐛 / ♻️ 등 | `✨ 홈 화면 레이아웃 구현` |
| CI·자동화 워크플로 | 👷 | `👷 PR 본문 자동화 워크플로 추가` |
| 스킬 추가/수정 | 👩‍💻 | `👩‍💻 리뷰 스킬 추가` |
| 하네스 규칙 수정 | 🔧 | `🔧 커밋 규칙 AGENTS.md에 추가` |

## 커밋 순서

하나의 사이클 안에서 여러 종류의 변경이 있을 때, 아래 순서로 커밋을 분리한다:

1. 📝 기획서 / 스펙 / ADR
2. ✨🐛♻️ 구현
3. 👷 CI·자동화 워크플로
4. 👩‍💻 스킬
5. 🔧 AGENTS.md / .codex/codex.md

## 커밋 분리 규칙

- 스킬(👩‍💻)과 AGENTS.md / .codex/codex.md(🔧)는 반드시 **별도 커밋**으로 분리한다
- 스킬은 **기능별로 커밋을 분리**한다 (예: 리뷰 스킬과 커밋 스킬은 별도 커밋)
- 같은 gitmoji 카테고리라도 피처가 다르면 분리한다
- 공백 보완으로 생긴 `[제안]` 문서 변경(📝)은 구현(✨)과 **별도 커밋**으로 분리한다 (둘 다 같은 브랜치, ADR-002 "기록=반영")

## 절차

1. `git status`로 변경된 파일을 확인한다
2. 변경 내용을 위 커밋 단위 기준으로 그룹핑한다
3. 커밋 순서에 맞게 순차적으로 커밋한다
4. 각 커밋마다 관련 파일만 `git add`하여 스테이징한다
