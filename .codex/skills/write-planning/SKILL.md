# 기획서 작성 스킬

## 트리거

민서가 기획서 작성을 요청하거나, Confluence 링크/내용을 전달했을 때 사용한다.

## 입력

민서가 아래 중 하나 이상을 제공한다:

- Confluence 기획서 링크 또는 본문 내용
- 피처명
- 구두 설명

## 작성 절차

1. 민서가 제공한 기획 내용을 정리한다
2. 작성할 문서의 **분류 위치**를 결정한다
   - 여러 페이지에서 공통으로 쓰이는 요소 → `docs/planning/common/`
   - 특정 페이지 전용 요소 → `docs/planning/<페이지명>/` (예: `docs/planning/home/filter.md`)
   - 분류가 모호하면 임의 결정하지 말고 민서에게 질문한다
3. **결정한 분류 폴더 안에서** 기존 파일 번호를 확인하여 다음 번호를 부여한다 (예: `home/01-filter.md` 다음은 `home/02-list.md`)
4. `docs/planning/00-template.md`를 복사해 그 구조에 맞춰 기획서를 작성한다
5. 작성 완료 후 민서에게 리뷰를 요청한다

## 문서 구조

기획서 구조의 **단일 원천**은 `docs/planning/00-template.md`다. 이 파일을 복사해 작성하며, 구조(섹션·표·`변경 이력` 양식)를 이 스킬에 중복 정의하지 않는다 (ADR-002 단일 원천).

## 상태 관리

- 최초 작성 시 상태를 `초안`으로 설정한다
- 민서가 피드백을 주면 반영 후 `리뷰 중`으로 변경한다
- 민서가 확정 의사를 밝히면 `**확정됨**`으로 변경한다
- 상태가 `**확정됨**`이 아니면 다음 단계(FE 스펙 작성)로 넘어갈 수 없다

## 규칙

- 민서가 제공하지 않은 내용을 임의로 추가하지 않는다
- 모호한 부분은 `[TODO: 민서 확인 필요]`로 표시하고 리뷰 시 질문한다
- 작성일은 오늘 날짜, 작성자는 항상 `민서`로 기입한다
- 파일명은 `XX-피처명.md` 형식을 따른다
- 번호는 **분류 폴더 내부 순번** 기준으로 매긴다 (예: `common/01-header.md`, `home/01-filter.md`, `home/02-list.md`)
- 즉, 서로 다른 폴더에 같은 번호가 존재할 수 있으며 이는 정상이다
