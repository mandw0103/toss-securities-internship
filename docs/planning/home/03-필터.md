## 개요

홈 화면 차트 섹션의 정렬·필터 컨트롤. 시장 구분 / 정렬 기준 / 기간 세 그룹으로 구성되며, 각 그룹은 항상 1개만 선택된다. 선택 변경 시 [04-리스트.md](./04-리스트.md)가 즉시 재구성된다.

## 사용자 시나리오

1. 사용자가 홈(/)에 진입하면 [02-탭.md](./02-탭.md) 아래에 세 그룹의 필터 버튼이 노출되며, 각 그룹의 가장 좌측 항목이 기본 활성 상태이다
2. 사용자가 그룹 내 다른 항목을 클릭하면 흰 박스(슬라이딩 인디케이터)가 해당 위치로 이동하고, 선택된 항목 텍스트가 볼드 처리된다
3. 그룹 내 다른 항목으로 변경하면 즉시 리스트가 재구성된다

## 화면 구성

### 화면 1: 필터

세 그룹으로 구성되며, 그룹별로 항상 1개만 선택된다.

- **좌측 그룹 (시장 구분, 3개)**: `전체` / `국내` / `해외`
  - 기본값: `전체`
- **중앙 그룹 (정렬 기준, 6개)**: `토스증권 거래대금` / `토스증권 거래량` / `거래대금` / `거래량` / `급상승` / `급하락`
  - 기본값: `토스증권 거래대금`
- **우측 그룹 (기간, 7개)**: `실시간` / `1일` / `1주일` / `1개월` / `3개월` / `6개월` / `1년`
  - 기본값: `실시간`
- 선택 시 흰 박스(슬라이딩 인디케이터)가 해당 위치로 이동하고, 선택된 항목 텍스트는 볼드 처리된다

## 데이터 요구사항

별도 서버 데이터 없음. 클라이언트 상태로 세 그룹의 현재 선택값만 보유한다.

| 항목 | 타입 | 설명 |
|------|------|------|
| selectedMarket | `'전체' \| '국내' \| '해외'` | 좌측 그룹 선택 |
| selectedSort | `'토스증권 거래대금' \| '토스증권 거래량' \| '거래대금' \| '거래량' \| '급상승' \| '급하락'` | 중앙 그룹 선택 |
| selectedPeriod | `Period` (= `'실시간' \| '1일' \| '1주일' \| '1개월' \| '3개월' \| '6개월' \| '1년'`) | 우측 그룹 선택 |

`Period` 타입은 [common/02-주식데이터.md](../common/02-주식데이터.md)에 정의된 것을 그대로 사용한다.

## 비즈니스 규칙

- 세 그룹은 서로 독립적이며, 각 그룹에서 항상 정확히 1개가 선택된 상태를 유지한다
- 모든 그룹의 기본 활성값은 **가장 좌측 항목**이다
- 어떤 그룹의 선택이라도 변경되면 즉시 [04-리스트.md](./04-리스트.md)가 재정렬·재필터링된다 (구체적 매핑은 04 문서 참조)
- **이번 범위에서 의도적으로 제외한 항목**: 실제 토스증권 사이트의 차트 섹션 필터 영역에는 "투자위험 주식 숨기기" 토글이 있지만, Confluence 기획서에 명시되지 않았으므로 이번 범위에서는 구현하지 않는다.

## 인터랙션 요약

| 트리거 | 동작 |
|--------|------|
| 시장 필터 클릭 | 흰 박스 이동 + 볼드 처리, 리스트가 해당 시장으로 필터링됨 |
| 정렬 기준 필터 클릭 | 흰 박스 이동 + 볼드 처리, 리스트가 해당 기준으로 재정렬됨. 거래대금/거래량 계열은 6번째 컬럼 헤더/값 변경, 등락률 계열(`급상승`/`급하락`)은 컬럼 개수 자체가 8→7로 바뀜 (자세한 매핑은 [04-리스트.md](./04-리스트.md) 참조) |
| 기간 필터 클릭 | 흰 박스 이동 + 볼드 처리, 리스트가 해당 기간의 `metricsByPeriod` 값을 기준으로 재정렬됨 |
