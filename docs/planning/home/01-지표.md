## 개요

홈 화면 상단에서 사용자가 빠르게 시장 상황을 파악할 수 있도록 **장 운영 상태 표시**와 **지표 캐러셀(증시 캘린더 / 환율 / 추천 주식)**을 제공한다. 둘 다 시장 상태를 빠르게 인지시키는 성격의 컴포넌트이므로 한 문서에서 다룬다.

## 사용자 시나리오

1. 사용자가 홈(/)에 진입한다
2. 화면 좌측 상단에 `국내 장 닫힘`, `해외 장 닫힘` 두 텍스트가 노출된다
3. 그 아래에 증시 캘린더 카드, 환율 카드, 추천 주식 카드가 가로로 나열된 캐러셀이 노출된다
4. 사용자가 우측 화살표 버튼을 클릭하면 다음 카드들이 보이도록 캐러셀이 좌측으로 이동한다
5. 사용자가 증시 캘린더 카드를 클릭하면 증시 페이지로 이동(이번 범위 외, 라우트만 정의)
6. 사용자가 환율 카드를 클릭하면 환율 페이지로 이동(이번 범위 외, 라우트만 정의)
7. 사용자가 추천 주식의 "주식명+그래프" 영역을 클릭하면 해당 주식 상세 페이지로 이동(이번 범위 외, 라우트만 정의)
8. 사용자가 "실시간 시세 확인하기" 버튼을 클릭하면 로그인 페이지로 이동(이번 범위 외, 라우트만 정의)

## 화면 구성

### 화면 1: 장 운영 상태 표시

- 화면 좌측 상단에 **두 텍스트**가 동시에 노출된다.
  - `국내 장 닫힘`
  - `해외 장 닫힘`
- 토스 원본을 따른 정적 텍스트이며, 이번 범위에서는 시각·상태 변동을 반영하지 않는다.

### 화면 2: 지표 캐러셀

- **컨테이너**: 가로 스크롤 가능한 카드 리스트
- **카드 종류**
  - 증시 캘린더 카드 (1개)
  - 환율 카드 (1개)
  - 추천 주식 카드 (약 3개) — 각 카드는 `주식명 + 그래프 + 실시간 시세 확인하기 버튼` 구성
- **캐러셀 화살표 버튼**
  - 카드 영역 좌·우 끝에 반투명 원형 버튼으로 노출
  - 클릭 시 **한 화면 단위**로 좌·우 이동
  - 캐러셀이 가장 좌측에 있으면 좌측 화살표는 노출하지 않는다
  - 캐러셀이 가장 우측에 있으면 우측 화살표는 노출하지 않는다

## 데이터 요구사항

| 항목 | 타입 | 설명 | 비고 |
|------|------|------|------|
| 장 운영 상태 텍스트 | 정적 문자열 2개 | `국내 장 닫힘`, `해외 장 닫힘` | 이번 범위에서 동적 갱신 없음 |
| 증시 캘린더 | 정적 카드 | 클릭 시 `/calendar` 등으로 이동 | 페이지 자체는 미구현 |
| 환율 | 정적 카드 | 클릭 시 `/exchange` 등으로 이동 | 페이지 자체는 미구현 |
| 추천 주식 목록 | `Stock` 풀에서 선정한 약 3건 + 미니 그래프 데이터 | 종목 데이터 정의는 [common/02-주식데이터.md](../common/02-주식데이터.md) 참조 | 선정 기준·sparkline 처리는 공통 문서의 TODO 참조 |

## 비즈니스 규칙

- 추천 주식 카드 개수는 약 3개를 기본으로 한다
- 카드 클릭 시 이동 페이지는 **이번 범위에서 구현하지 않으며**, 라우트 이동만 동작하도록 한다
- "실시간 시세 확인하기" 버튼은 추천 주식 카드 내에서 별도 클릭 영역으로 분리되어, 카드 본문 클릭과 다른 페이지(로그인)로 이동한다

## 예외 처리

| 상황 | 처리 방식 |
|------|-----------|
| 추천 주식 데이터가 비어 있음 | 캐러셀에 증시 캘린더 / 환율 카드만 노출 |
| 카드가 한 화면에 모두 들어옴 | 좌·우 화살표 모두 미노출 |
