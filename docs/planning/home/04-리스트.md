## 개요

홈 화면 차트 섹션 하단의 종목 리스트. [03-필터.md](./03-필터.md)의 시장/정렬/기간 선택값에 따라 100개 종목을 정렬·필터링하여 노출하며, 정렬 기준에 따라 컬럼 구성이 동적으로 달라진다. 데이터 정의(Stock 구조, mock 전략)는 [common/02-주식데이터.md](../common/02-주식데이터.md) 참조.

## 사용자 시나리오

1. 사용자가 홈(/)에 진입하면 필터 기본값(`전체` / `토스증권 거래대금` / `실시간`) 기준으로 정렬된 100개 종목이 리스트로 노출된다
2. 사용자가 필터를 변경하면 리스트가 즉시 재구성된다
3. 사용자가 리스트의 종목 행을 클릭하면 해당 주식 상세 페이지로 이동(이번 범위 외)

## 화면 구성

리스트는 테이블/리스트 형태이며, **컬럼 구성은 정렬 기준에 따라 동적으로 달라진다.**

### 화면 1-A. 거래대금/거래량 계열 정렬일 때 (8개 컬럼)

해당 정렬: `토스증권 거래대금` / `토스증권 거래량` / `거래대금` / `거래량`

| 컬럼 | 설명 |
|------|------|
| 순위 | 현재 정렬 결과의 인덱스+1로 산출되는 **파생값** (데이터에 저장하지 않음) |
| 로고 | 종목명 첫 글자 + 시드 기반 랜덤 배경색의 간이 로고 (`Stock.logoSeed`) |
| 종목명 | `Stock.name` 텍스트 |
| 현재가 | `Stock.currentPrice`. 기간 무관 고정값. 표기는 시장 구분 없이 항상 **`₩` 접두** + 천 단위 콤마 (예: `₩78,900`, `₩245.32`) |
| 등락률 | 선택한 기간의 등락률(%) — `metricsByPeriod[period].changeRate`. 양수 빨강·음수 파랑 등 색상 구분 |
| 6번째 컬럼 (정렬 기준에 따라 헤더·값이 바뀜) | 정렬 기준명을 그대로 헤더로 사용한다. `토스증권 거래대금 순` / `토스증권 거래량 순` / `거래대금 순` / `거래량 순` 중 하나. 값은 그 기준에 해당하는 metric (예: `토스증권 거래대금 순` → `metricsByPeriod[period].tossTradingValue`) |
| 토스증권 거래 비율 | `Stock.tossSharePercent` (%) |
| 토스증권 AI 요약 | `Stock.aiSummary` 한 줄 텍스트 |

### 화면 1-B. 등락률 계열 정렬일 때 (7개 컬럼)

해당 정렬: `급상승` / `급하락`

| 컬럼 | 설명 |
|------|------|
| 순위 | 위와 동일 |
| 로고 | 위와 동일 |
| 종목명 | 위와 동일 |
| 현재가 | 위와 동일 |
| 등락률 순 | 선택한 기간의 등락률(%) — `metricsByPeriod[period].changeRate`. 5번째 컬럼이 "등락률 순"이 되어, 화면 1-A에 있던 별도의 6번째 컬럼은 사라진다 |
| 토스증권 거래 비율 | 위와 동일 |
| 토스증권 AI 요약 | 위와 동일 |

### 공통

- 종목 행 클릭 시 주식 상세 페이지로 이동(이번 범위 외)
- 행 개수: **100개 고정**, 더보기/페이지네이션 없음

## 비즈니스 규칙

- 시장 필터(`국내`/`해외`)가 적용되면 해당 시장의 종목만 리스트에 노출한다 (`전체`는 필터링 없음)
- 정렬은 다음 매핑에 따른다:
  - `토스증권 거래대금` → `metricsByPeriod[period].tossTradingValue` 내림차순
  - `토스증권 거래량` → `metricsByPeriod[period].tossTradingVolume` 내림차순
  - `거래대금` → `metricsByPeriod[period].totalTradingValue` 내림차순
  - `거래량` → `metricsByPeriod[period].totalTradingVolume` 내림차순
  - `급상승` → `metricsByPeriod[period].changeRate` 내림차순
  - `급하락` → `metricsByPeriod[period].changeRate` 오름차순
- 정렬 기준에 따라 컬럼 구성이 동적으로 변한다 (화면 1-A / 1-B 참조)
- 순위는 시장/정렬/기간 적용 후의 정렬된 배열에서 인덱스+1로 산출하며, 데이터에 저장하지 않는다

## 예외 처리

| 상황 | 처리 방식 |
|------|-----------|
| 시장 필터 결과가 0건 | 별도 안내 없이 빈 리스트(빈 화면)로 노출 |
| mock 데이터 로드 실패 | 이번 범위에서는 발생하지 않음 (정적 import) |
