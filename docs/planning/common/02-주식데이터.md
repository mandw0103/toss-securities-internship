## Stock 데이터 구조

```ts
type Period = '실시간' | '1일' | '1주일' | '1개월' | '3개월' | '6개월' | '1년'

interface PeriodMetrics {
  changeRate: number          // 등락률(%) — 리스트 등락률 컬럼 + 급상승/급하락 정렬용
  tossTradingValue: number    // 토스증권 거래대금
  tossTradingVolume: number   // 토스증권 거래량
  totalTradingValue: number   // 전체 거래대금
  totalTradingVolume: number  // 전체 거래량
}

interface Stock {
  stockId: string
  name: string                              // 종목명
  market: '국내' | '해외'                    // 시장 구분 (좌측 필터)
  logoSeed: string                          // 보통 name의 첫 글자. 색상 hash 입력값
  currentPrice: number                      // 현재가 (기간 무관)
  tossSharePercent: number | null           // 토스증권 거래 비율. API 미제공 시 빈 값(null)
  aiSummary: string                         // 토스증권 AI 요약 한 줄
  metricsByPeriod: Record<Period, PeriodMetrics>
}
```

- 정렬 기준 6 × 기간 7 조합이 모두 자연스럽게 매핑된다.
  - 예: `정렬=거래대금` + `기간=1주일` → `metricsByPeriod['1주일'].totalTradingValue` 내림차순
  - 예: `정렬=급상승` → `metricsByPeriod[period].changeRate` 내림차순 / `급하락` → 오름차순

### 추천 주식 카드(지표 캐러셀)의 미니 그래프(sparkline)

- **Stock 필드에 sparkline을 두지 않는다.**
- 추천 카드 렌더 시점에 PRNG(시드: `stockId`)로 시계열 시세 포인트 **약 30개**를 즉석 생성한다.
- sparkline 생성 함수는 `src/mocks/stocks.ts` 내부 유틸로 두며, 시그니처는 대략 `generateSparkline(stockId: string, length?: number): number[]` 형태.
- 결정 근거: 미래 주식 상세 페이지의 가격 차트는 길이·해상도·필드(OHLCV 등)가 sparkline과 달라 별도 구조가 필요하므로, 지금 Stock에 박아둬도 재활용성이 낮음.

## Mock 데이터 전략

- **저장 위치**: `src/mocks/stocks.ts`
- **TanStack Query 호환 wrapping**: `fetchStocks(): Promise<Stock[]>` 비동기 함수로 감싼다. 추후 실 API로 교체 시 호출부 수정 없음.
- **종목 메타 (100건)**:
  - Claude가 실제로 거래되는 종목들을 참고해 직접 작성한다 (실제 데이터와 완전 동일할 필요는 없고 임의 수정 허용).
  - 구성: **국내 70건** (KOSPI/KOSDAQ 대표 종목들), **해외 30건** (NYSE/NASDAQ 대표 종목들).
  - 각 종목의 메타 필드: `stockId`, `name`, `market`, `logoSeed`, `currentPrice`, `tossSharePercent`, `aiSummary`까지 직접 작성.
  - `aiSummary`는 종목별로 임의의 한 줄 텍스트(예: `1분기 실적 발표`, `인력감축&가이던스 실망`, `상장 유지 실패`, `매출 감소` 등)에서 골라 배정. Claude가 종목 성격에 맞춰 임의 작성·재사용 가능.
- **`metricsByPeriod` 생성 (B-1 채택)**:
  - 손으로 100 × 7 × 5 = 3,500개 값을 적는 대신, **시드 고정 의사난수(PRNG) 함수**로 모듈 로드 시 1회 생성한다.
  - 시드를 종목별로 고정 (예: `stockId` 기반)하여 새로고침해도 항상 동일한 값을 산출한다.
  - 외부 라이브러리는 사용하지 않고 **인하우스 구현**으로 간다 (예: `mulberry32` 등 5~10줄짜리 짧은 PRNG 함수를 `src/mocks/` 내부 유틸로 둔다).
- **추천 주식 선정**: Stock 풀 100건에서 **`metricsByPeriod['1일'].tossTradingValue` 기준 상위 3건**을 동적으로 선정해 추천 종목으로 노출한다 (= 토스증권 1일 거래대금 상위 3등). 시장 구분은 `전체` 풀 기준.
- **로고**: `logoSeed`(보통 name의 첫 글자) + 시드 기반 해시 색상으로 컴포넌트 내에서 즉석 렌더링. 외부 이미지 URL/CDN 사용하지 않음.

## 이슈 13 - 홈 실시간 차트 API 연동 범위

- 대상 API는 토스증권 홈의 실시간 차트 랭킹 요청 1개로 한정한다.
- 요청 URL: `POST https://wts-cert-api.tossinvest.com/api/v2/dashboard/wts/overview/ranking`
- 요청 payload는 아래 값으로 고정한다.

```json
{
  "id": "biggest_total_amount",
  "filters": [
    "KRX_MANAGEMENT_STOCK",
    "MARKET_CAP_GREATER_THAN_50M",
    "STOCKS_PRICE_GREATER_THAN_ONE_DOLLAR"
  ],
  "duration": "realtime",
  "tag": "all"
}
```

- SSE(Server-Sent Events), polling, 주기적 재조회는 이번 범위에서 제외한다.
- 클론 사이트 최초 진입 시 기본 HTTP 요청으로 한 번 받아오는 흐름에 집중한다.
- 브라우저 CORS 정책에 대응하기 위해 로컬 개발 환경에서는 Vite dev proxy를 사용한다.
- 응답의 `result.products`를 홈 차트 리스트의 원천 데이터로 사용한다.
- API 응답에 시장 구분 필드가 없으므로 `productCode` prefix로 시장을 추정한다. `US`, `NAS`, `NYS`, `AMX` prefix는 `해외`로 보고, 그 외 한국 종목 코드 패턴은 `국내`로 본다.
- API 응답에 없는 `tossSharePercent`(토스증권 거래 비율), `aiSummary`(토스증권 AI 요약)는 빈 값으로 둔다.
- 검색 API 연동은 이번 범위에서 제외하고 기존 mock 검색을 유지한다.
