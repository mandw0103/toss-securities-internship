# ADR-002: 기획-개발 싱크 시스템 설계

> 작성일: 2026-05-28
> 상태: 제안됨 | **채택됨** | 폐기됨
> 관련 이슈: #12 (기획-개발 싱크 문제 해결)
> 관련 ADR: [ADR-001](01-pr-body-automation.md) (PR 본문 자동화 — 게이트가 올라탈 레일)

## 맥락

- **문제**: 기획서 기반 개발 중 코드는 수정됐지만 기획서엔 반영되지 않는 불일치가 생긴다. 특히 **개발자가 기획의 빈 부분을 구현으로 보완했지만 그 내용이 기획서에 반영되지 않는 경우**가 이번 목표다.
- **설계 원칙**: 사람이 잘해서 막는 게 아니라, **실수해도 어긋나지 않는 시스템**(poka-yoke)을 만든다. 사람 개입은 최소화하고 AI 자동화를 최대화한다.
- **불일치 4종** (각 장치가 무엇을 막고 못 막는지 정직하게 보기 위함):
  1. 공백 보완 미반영 — 코드 O / 문서 X (이번 주 목표)
  2. 의도 변경 미반영 — 코드 O / 문서 X
  3. 기획만 바뀌고 코드 미반영 — 문서 O / 코드 X
  4. 둘 다 안 건드렸는데 의미가 어긋남 — 둘 다 X
- **진단**: 현 시스템은 감지(implement의 "임의 판단", review의 "기획서/스펙 문제" 분류)는 되지만, **역반영(문서 수정)이 사람 핸드오프**라 거기서 누락된다.

## 고려한 선택지

### 선택지 1: 사람 규율 (체크리스트·리뷰만)
- 장점: 도입이 쉽다
- 단점: 체크박스는 무의식적으로 눌린다 → "실수해도 안 어긋남"을 만족하지 못한다

### 선택지 2: 단일 진실원천만으로 해결
- 장점: 어긋날 자리 자체를 없앤다
- 단점: 개요·시나리오 같은 prose는 단일원천화가 불가능해 부분 해결에 그친다

### 선택지 3: 자동 게이트 + 기록=반영을 핵심으로, 단일원천·ID를 보조로 (계층 설계)
- 장점: 강제는 기계가, 의미 판단은 AI가, 어긋날 자리는 단일원천이 제거 — 계층이 서로의 약점을 보완한다
- 단점: CI·권한 인프라가 필요하고 일부는 점진 도입이 필요하다

## 결정

**선택지 3을 채택한다.** 아래 5계층을 우선순위에 따라 단계적으로 도입한다.

### 중요
**1. "기록 = 반영"을 하나로 묶기** **[개발 완료]**
- 공백을 보완하면 따로 메모하지 않고 **기획서 자체를 같이 고쳐 같은 PR에** 넣는다. 메모와 반영이 한 몸이라 "기록했는데 반영을 깜빡"이 구조적으로 불가능해진다.
- implement 스킬이 공백 보완 시 대상 기획서/스펙의 **"변경 이력" 표에 `[제안]` 행 + 패치 diff**를 같은 브랜치에 만든다. 이 diff가 곧 2번 게이트의 "docs 변경됨" 증거다. 머지되면 `[제안]`을 `[확정]`으로 전환한다(2번 게이트 도입 시 자동, 그 전엔 수동).

**2. 자동 검문소 (단일 게이트)** **[부분 개발 — 2a 완료 · 2b 생략]**
- 문서와 코드가 안 맞으면 머지 자체가 안 되게 막는다. 사람이 깜빡해도 통과가 안 된다.
- GitHub Branch Protection으로 main 머지를 막고, GitHub Actions가 PR마다 두 검사를 돌려 통과해야 풀어준다.
  - **결정적 검사(기계)**: `src/**`는 바뀌었는데 `docs/**`는 안 바뀌었고 `no-spec-change` 라벨도 없으면 → 실패 — **[2a 개발 완료]** (`.github/workflows/sync-gate.yml`; main Branch Protection required check 등록은 별도 admin 작업)
    - 한글 파일명 경로가 `git diff --name-only`에서 quote/escape 처리되면 `docs/**` 감지가 실패할 수 있으므로, 게이트는 `git -c core.quotePath=false diff --name-only`로 변경 파일을 읽는다.
  - **AI 검사(현재 사용 AI 도구)**: PR diff를 읽어 스펙에 없는 동작·빠진 예외 처리가 있으면 경고 코멘트, 붙은 라벨이 정당한지도 감사한다. 현재 사용하는 도구가 Claude면 Claude를, Codex면 Codex를 사용한다. — **[2b 개발 생략]** (1인 리뷰 체제가 커버하여 인턴십 범위에서 제외; 기여자 증가·라벨 남용 시 재고)



### 권장

**3. 단일 진실원천** **[개발 생략 — 인턴십 범위 외]**
- 기계가 읽을 수 있는 것만 단일원천으로 뺀다.
  - API 타입 = 스펙의 TS `interface`를 코드가 직접 `import`
  - 라우팅 경로 = 상수 한 곳에 정의
  - 필터칩·GNB 탭·정렬 옵션 같은 **설정성 데이터 = 민서가 JSON으로 직접 작성·커밋, 코드가 로드** (JSON이 곧 기획서이자 코드 — 민서가 코드는 못 짜도 JSON은 소유 가능)
- 개요·시나리오 같은 prose는 단일원천이 불가능하므로 2번 게이트와 AI 리뷰로만 잡는다.

**4. 트레이서빌리티 ID** **[개발 생략 — 인턴십 범위 외]**
- 기획서 표의 각 행에 ID(예: `BR-HOME-FILTER-01`)를 주고 코드/테스트가 참조한다.
  - **정방향(문서→코드)**: grep으로 "기획엔 있는데 코드에 없는 ID" = 미구현 탐지 (기계)
  - **역방향(코드→문서)**: "ID로 설명 안 되는 동작" = 미문서화 탐지 (정규식 불가 → AI가 처리, 2번 AI 검사를 더 정확하게 함)

### 선택

**5. 원본 기획(Confluence)과의 동기화** **[개발 생략 — 인턴십 범위 외]**
- 웹훅 수신 서버 대신(인턴 프로젝트엔 과함) 스케줄된 현재 사용 AI 도구 + Atlassian MCP가 Confluence ↔ `docs/planning`을 비교(in)하고, 머지 시 AI가 Confluence 상태 변경을 **제안**한다. 현재 사용하는 도구가 Claude면 Claude를, Codex면 Codex를 사용한다(정본 자동 쓰기는 위험 → 사람 확인 후).
- 단, 이는 "코드가 공백을 메웠는데 미반영"과는 **다른 종류의 불일치**라 분리해서 본다.

## 표준 사이클 (1·2가 맞물리는 흐름)

1. 구현 중 기획에 없던 케이스 발견 → 코드로 보완
2. implement가 같은 브랜치에서 기획서 "변경 이력"에 `[제안]` 행 + diff 생성 (= 기록이자 반영)
3. PR 올림 → Actions가 "src 바뀜 + docs 바뀜 OK" 확인, AI가 "스펙 외 동작 없음" 확인
4. 민서는 PR에서 문서 diff를 인라인으로 보고 **승인만** 함
5. 머지 → `[제안]`을 `[확정]`으로 전환 (2번 게이트 도입 시 자동, 그 전엔 수동)

## ADR-001(PR 본문 자동화)와의 관계·주의

- **마커로 자동/수동 영역 분리**: 워크플로·게이트는 `<!-- AUTO:...:start/end -->` 마커 **안**만 갱신하고, Summary·"기획 변경 여부" 같은 마커 **밖** 작성자 텍스트는 절대 건드리지 않는다.
- **치명도(criticality) 분리**: 두 자동화를 한 워크플로·한 실패 의미로 뭉치면 충돌한다. 반드시 분리한다.

  | | 실패 시 | status check |
  |---|---|---|
  | #14 (PR 본문 작성) | "수동 입력해주세요" 안내만, 머지 막지 않음 | **비차단(advisory)** |
  | 게이트 (싱크 판정) | 머지 차단 | **required check** |

- 게이트는 #14 워크플로의 브랜치/커밋 파싱·플러밍 레일을 재사용한다.

## 분류 규칙

- **도구/CI/자동화 작업은 `docs/planning`·`docs/spec`(FE 피처용)이 아니라 ADR(`docs/adr`)로 기록한다.** 이 규칙은 현재 사용하는 AI 도구의 운영 규칙 문서에 명시한다. Claude를 사용하면 `CLAUDE.md`에, Codex를 사용하면 `AGENTS.md`와 `.codex/codex.md`에 반영한다 (item 1 작업에 포함, 반영 완료).

## 결과

- 사람의 일이 "작성"에서 "승인"으로 축소되고, 누락은 게이트가 차단한다.
- **커버리지(정직하게)**: 게이트 = 불일치 1·2 / ID = 3 / 테스트·단일원천 = 4. 게이트 단독으로는 4(둘 다 미편집)를 못 잡는다.
- **한계**: 게이트는 "문서를 안 건드림"만 잡고 "틀리게 건드림"은 AI 리뷰 몫이다. 라벨 우회는 AI가 감사한다. ID는 정방향 grep으로 자정한다.
