## 타입 정의

```ts
// src/types/stock.ts
export type Period = '실시간' | '1일' | '1주일' | '1개월' | '3개월' | '6개월' | '1년';

export type Market = '국내' | '해외';

export interface PeriodMetrics {
  changeRate: number;          // 등락률(%)
  tossTradingValue: number;    // 토스증권 거래대금
  tossTradingVolume: number;   // 토스증권 거래량
  totalTradingValue: number;   // 전체 거래대금
  totalTradingVolume: number;  // 전체 거래량
}

export interface Stock {
  stockId: string;
  name: string;
  market: Market;
  logoSeed: string;
  currentPrice: number;
  tossSharePercent: number | null;
  aiSummary: string;
  metricsByPeriod: Record<Period, PeriodMetrics>;
}

export const PERIODS: Period[] = ['실시간', '1일', '1주일', '1개월', '3개월', '6개월', '1년'];

// 검색 결과는 풀 Stock의 부분 집합 — 헤더 검색이 사용
export type StockSearchResult = Pick<Stock, 'stockId' | 'name'>;
```

> `src/types/stock.ts`는 본 스펙에서 신규 생성하며, 헤더 검색([common/01-header.md](./01-header.md))도 동일 파일에서 `Stock`/`StockSearchResult`를 import 한다. 기존 `src/apis/stocks.ts`의 로컬 `Stock { id, name }` 정의는 제거되고 검색 응답이 `StockSearchResult`로 갱신된다.

## Mock 모듈

### PRNG 유틸 (`src/mocks/prng.ts`)

```ts
export function mulberry32(seed: number): () => number;
export function hashStringToSeed(input: string): number;
```

**동작:**
- `mulberry32(seed)`: 5~10줄짜리 인하우스 PRNG. `0 <= x < 1`을 반환하는 함수를 돌려준다.
- `hashStringToSeed(input)`: 문자열을 32-bit 정수 시드로 변환 (예: FNV-1a). `stockId` → 시드 변환에 사용한다.
- 외부 라이브러리 사용 금지.

### 종목 메타 (`src/mocks/stocks.ts`)

- 모듈 내부 상수로 100건의 종목 메타 배열을 직접 작성한다.
  - 국내 70건 (KOSPI/KOSDAQ 대표 종목)
  - 해외 30건 (NYSE/NASDAQ 대표 종목)
- 각 항목에 작성할 필드: `stockId`, `name`, `market`, `logoSeed`, `currentPrice`, `tossSharePercent`, `aiSummary`.
- `aiSummary`는 종목 성격에 맞는 한 줄 텍스트(예: `1분기 실적 발표`, `매출 감소`, `상장 유지 실패`)에서 임의 배정.

### `metricsByPeriod` 생성

- 종목 메타에는 `metricsByPeriod`를 작성하지 않는다.
- 모듈 로드 시 1회, 각 종목별로 시드(`hashStringToSeed(stockId)`) 고정 PRNG로 7개 기간 × 5개 metric = 35개 값을 생성하여 결합한다.
- 결과는 `STOCKS: Stock[]` 상수로 보관. 새로고침해도 동일 값.

```ts
// src/mocks/stocks.ts
export const STOCKS: Stock[];
export function fetchStocks(): Promise<Stock[]>;
export function searchStocks(query: string): Promise<StockSearchResult[]>;
export function generateSparkline(stockId: string, length?: number): number[];
```

### Sparkline 생성

```ts
generateSparkline(stockId: string, length = 30): number[]
```

**동작:**
- 시드: `hashStringToSeed(stockId)` 기반.
- 기본 길이: 30.
- 추천 주식 카드가 렌더 시점에 호출하며, 동일 시드는 항상 동일 결과를 보장한다.
- `Stock` 필드에는 포함하지 않는다.

### `fetchStocks`

```ts
fetchStocks(): Promise<Stock[]>
```

**동작:** 이슈 13부터 토스증권 홈 실시간 차트 랭킹 API를 호출하고, 응답을 `Stock[]`로 매핑해 반환한다. API 호출 실패 시 fallback 없이 Query 에러 상태로 전달한다.

### `searchStocks`

```ts
searchStocks(query: string): Promise<StockSearchResult[]>
```

**동작:**
- `query`가 빈 문자열이면 빈 배열을 반환한다.
- 그 외에는 `STOCKS`에서 `name`이 `query`를 포함(대소문자 무시)하는 종목을 필터링하여 `{ stockId, name }`만 매핑해 반환한다.
- `Promise.resolve(...)`로 감싸 비동기 시그니처를 유지한다. 추후 실 API 교체 시 호출부 수정 없음.
- 사용처: [common/01-header.md](./01-header.md)의 헤더 검색 드롭다운.

## API 연동

### `fetchStocks(): Promise<Stock[]>`

**요청:** 토스증권 홈 실시간 차트 랭킹 API에 고정 payload로 `POST` 요청한다.

```ts
const TOSS_RANKING_ENDPOINT = '/toss-api/api/v2/dashboard/wts/overview/ranking';

const TOSS_RANKING_PAYLOAD = {
  id: 'biggest_total_amount',
  filters: [
    'KRX_MANAGEMENT_STOCK',
    'MARKET_CAP_GREATER_THAN_50M',
    'STOCKS_PRICE_GREATER_THAN_ONE_DOLLAR',
  ],
  duration: 'realtime',
  tag: 'all',
} as const;
```

**원본 응답:** `result.products` 배열

```ts
interface TossRankingResponse {
  result: {
    basedAt: string;
    type: 'STOCK';
    products: TossRankingProduct[];
    rankingIdWithDuration: string;
    rankingId: string;
    duration: 'realtime';
    filters: string[];
    tag: 'all';
  };
}

interface TossRankingProduct {
  rank: number;
  productCode: string;
  name: string;
  logoImageUrl?: string;
  price: {
    priceType: 'TOSS_SECURITIES';
    base: number;
    close: number;
    baseKrw: number;
    closeKrw: number;
    tossSecuritiesVolume: number;
    tossSecuritiesAmount: number;
  };
  extraInfo?: {
    tossSecuritiesBuy?: number;
    tossSecuritiesSell?: number;
  };
}
```

**응답 매핑:** `TossRankingProduct[]`를 기존 `Stock[]`로 변환한다.

| Stock 필드 | Toss API 원본 | 매핑 규칙 |
|------------|---------------|-----------|
| `stockId` | `productCode` | 그대로 사용 |
| `name` | `name` | 그대로 사용 |
| `market` | `productCode` | `US`, `NAS`, `NYS`, `AMX` prefix는 `해외`, 그 외 한국 종목 코드 패턴은 `국내`로 추정 |
| `logoSeed` | `name` | `name.slice(0, 1)` 사용 |
| `currentPrice` | `price.closeKrw` | 원화 현재가로 사용 |
| `tossSharePercent` | - | 빈 값 처리를 위해 `null` 사용 |
| `aiSummary` | - | 빈 문자열 `''` 사용 |
| `metricsByPeriod['실시간'].changeRate` | `price.baseKrw`, `price.closeKrw` | `(closeKrw - baseKrw) / baseKrw * 100` |
| `metricsByPeriod['실시간'].tossTradingValue` | `price.tossSecuritiesAmount` | 그대로 사용 |
| `metricsByPeriod['실시간'].tossTradingVolume` | `price.tossSecuritiesVolume` | 그대로 사용 |
| `metricsByPeriod['실시간'].totalTradingValue` | - | API 미제공. `tossSecuritiesAmount`와 동일값 사용 |
| `metricsByPeriod['실시간'].totalTradingVolume` | - | API 미제공. `tossSecuritiesVolume`와 동일값 사용 |
| 나머지 기간의 `metricsByPeriod` | - | 이번 API 범위에서 미제공. `실시간`과 동일값 사용 |

**CORS 대응:**
- 토스증권 응답 헤더의 `Access-Control-Allow-Origin`은 `https://www.tossinvest.com`으로 확인되었다.
- 로컬 앱(`http://localhost:5173`)에서 직접 호출하면 브라우저 CORS 정책에 막힐 수 있으므로, Vite dev proxy로 `/toss-api` prefix를 토스증권 API origin에 연결한다.
- 프론트 코드는 실제 origin 대신 `/toss-api/api/v2/dashboard/wts/overview/ranking`을 호출한다.
- 브라우저 요청의 로컬 `Origin`/`Referer`가 upstream으로 전달되면 토스 API가 403을 반환하므로, Vite proxy에서 upstream 요청 헤더의 `origin`/`referer`를 `https://www.tossinvest.com` 기준으로 설정한다.

**시장 구분 추정:**
- 토스증권 랭킹 API 응답에는 `국내`/`해외` 시장 구분 필드가 없다.
- `productCode`가 `US`, `NAS`, `NYS`, `AMX`로 시작하면 `해외`로 매핑한다.
- 그 외 한국 종목 코드 패턴은 `국내`로 매핑한다.
- 추정 불가한 값은 구현 안정성을 위해 `해외`로 둔다.

**범위 제외:**
- SSE(Server-Sent Events), polling, 주기적 refetch는 구현하지 않는다.
- `duration`, `tag`, `id`는 이번 사이클에서 동적으로 바꾸지 않는다.
- 검색 API 연동은 이번 사이클에서 제외한다.

**에러 처리:** HTTP 실패 또는 응답 구조 불일치 시 Query 에러 상태로 노출한다. 화면 fallback 문구는 사용처 스펙에서 정의한다.

### `searchStocks(query: string): Promise<StockSearchResult[]>`

**요청:** `query` (검색어)

**응답:** `StockSearchResult[]` (매칭되는 종목들의 `{ stockId, name }`)

**에러 처리:** 정적 import이므로 발생하지 않음. 헤더는 `enabled: query.length > 0`으로 빈 검색어에는 호출하지 않는다.

## 상태 관리

### 서버 상태 (TanStack Query)

`useStocks` 훅(`src/hooks/useStocks.ts`)으로 진입점을 단일화한다.

```ts
export function useStocks(): UseQueryResult<Stock[]>;
```

- Query Key: `['stocks']`
- Query Fn: `fetchStocks`
- Stale Time: `Infinity` (클론 사이트 최초 진입 시 받은 랭킹 값을 유지)
- `refetchOnWindowFocus`: `false`
- polling 없음
- 동일 키를 공유하므로 [home/01-지표.md](../home/01-지표.md)·[home/04-리스트.md](../home/04-리스트.md)에서 화면당 1회만 페치된다.

### 셀렉터 (`src/utils/stocks.ts`)

본 스펙에서 파일을 신규 생성하되, 셀렉터 함수는 사용처 스펙에서 정의한다.

| 함수 | 정의 스펙 | 용도 |
|------|-----------|------|
| `selectRecommended(stocks, count)` | [home/01-지표.md](../home/01-지표.md) | 추천 주식 상위 N건 |
| `applyChartFilter(stocks, filter)` | [home/04-리스트.md](../home/04-리스트.md) | 차트 리스트 시장·정렬·기간 적용 |
