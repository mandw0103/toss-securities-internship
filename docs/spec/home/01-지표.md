## 컴포넌트 명세

### HomePage

홈 화면의 컴포지션 루트. [common/02-주식데이터.md](../common/02-주식데이터.md)의 `useStocks()`를 **본 컴포넌트에서 1회 호출**하고, 결과를 자식 트리에 props로 흘려보낸다.

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| - | - | - | props 없음 |

**동작:**
1. `const { data: stocks = [] } = useStocks();`
2. `const recommended = useMemo(() => selectRecommended(stocks, 3), [stocks]);`
3. 다음 자식을 순서대로 렌더:
   - `<MarketStatusBar />`
   - `<IndicatorCarousel recommendedStocks={recommended} />`
   - `<ChartSection stocks={stocks} />` ([02-탭.md](./02-탭.md))

**왜 HomePage가 호출하는가:** `useStocks()`는 `IndicatorCarousel`과 `ChartSection` 양쪽에서 필요하다. 동일 키이므로 두 자식이 각자 호출해도 TanStack Query가 dedupe하지만, 호출 주체를 단일화하면 의존성 흐름이 명확해지고 추후 props drilling 리팩토링이 쉽다.

### MarketStatusBar

- props 없음.
- 화면 좌측 상단에 두 `MarketStatusItem`(`국내 장 닫힘`, `해외 장 닫힘`)을 가로로 배치.
- 정적 텍스트만 노출. 시각·상태 변동 없음.

### MarketStatusItem

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| label | string | O | 표시 텍스트 |

**동작:** 정적 텍스트 렌더링. 클릭/이동 없음.

### IndicatorCarousel

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| recommendedStocks | Stock[] | O | 추천 주식 목록 (보통 3건) |

**내부 상태:**
- `viewportIndex: number` — 현재 노출 페이지 인덱스 (0부터)

**동작:**
- 카드는 `[증시 캘린더, 환율, ...추천 주식]` 순으로 나열.
- `recommendedStocks`가 빈 배열이면 추천 주식 카드 영역은 렌더하지 않고 캘린더·환율 카드만 노출.
- 좌·우 화살표는 **한 화면 단위**로 `viewportIndex`를 ±1.
- `viewportIndex === 0`이면 좌측 화살표 미노출.
- `viewportIndex === maxIndex`이면 우측 화살표 미노출.
- 모든 카드가 한 화면에 들어오면 좌·우 화살표 모두 미노출.

> "한 화면 단위" 측정 방식(viewport width 기반 vs 카드 개수 단위 등)은 구현 자유도로 둔다. 사용자 관점에서는 클릭당 한 페이지가 이동하면 된다.

### CarouselArrow

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| direction | `'left' \| 'right'` | O | 방향 |
| onClick | `() => void` | O | 클릭 콜백 |

**동작:** 카드 영역 좌·우 끝에 반투명 원형 버튼으로 노출.

### MarketCalendarCard

- props 없음.
- 클릭 시 `useNavigate('/calendar')`.

### ExchangeRateCard

- props 없음.
- 클릭 시 `useNavigate('/exchange')`.

### RecommendedStockCard

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| stock | Stock | O | 추천 주식 |

**구성:** 상단에 `Logo + 종목명`, 중단에 `Sparkline`, 하단에 `RealtimeQuoteButton`.

**동작:**
- `Logo + 종목명 + Sparkline` 영역 클릭 시 `useNavigate('/stocks/{stock.stockId}')`.
- `RealtimeQuoteButton` 클릭 시 `useNavigate('/login')`. 카드 본문 클릭 핸들러로 이벤트 전파를 차단.

### Logo (공통 컴포넌트)

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| logoSeed | string | O | 보통 종목명 첫 글자 |

**동작:** `logoSeed`의 첫 글자를 보여주고, `hashStringToSeed(logoSeed)` 기반 색상으로 배경을 채운다. [home/04-리스트.md](./04-리스트.md)와 공유.

### Sparkline

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| stockId | string | O | sparkline 시드 |
| length | number | X | 포인트 개수 (기본 30) |

**동작:**
- 마운트 시 `generateSparkline(stockId, length)`로 데이터 생성.
- 인라인 SVG 폴리라인으로 렌더 (외부 차트 라이브러리 사용 금지).

**색상 규칙:**
- 시작점(`points[0]`) 대비 종점(`points[length-1]`)이 **상승 또는 동일**하면 등락률 양수 색상(`#f04452`, 빨강)으로 그린다.
- 종점이 **하락**하면 등락률 음수 색상(`#3182f6`, 파랑)으로 그린다.
- 색상은 [04-리스트.md](./04-리스트.md)의 등락률 색상 규칙과 동일한 색을 사용해 화면 전체 일관성을 유지한다.

### RealtimeQuoteButton

## 상태 관리

### 서버 상태

- `HomePage`에서 `useStocks()`를 **1회** 호출 → `Stock[]` 구독.
- 추천 주식은 `HomePage`에서 `selectRecommended(stocks, 3)`로 파생하여 `IndicatorCarousel`에 props로 전달.
- [02-탭.md](./02-탭.md)·[04-리스트.md](./04-리스트.md)도 같은 `stocks`를 props로 받는다 (`ChartSection` 경유).

### 셀렉터 (`src/utils/stocks.ts`)

```ts
export function selectRecommended(stocks: Stock[], count = 3): Stock[];
```

**동작:**
- 시장 필터 없이 전체 풀 기준.
- `metricsByPeriod['1일'].tossTradingValue` 내림차순 정렬 후 상위 `count`개 반환.

### 클라이언트 상태

- `IndicatorCarousel` 내부 `useState<number>` (`viewportIndex`).
