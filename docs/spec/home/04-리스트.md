## 개요

홈 차트 섹션 하단의 종목 리스트. [03-필터.md](./03-필터.md)의 시장/정렬/기간 선택값에 따라 100개 종목을 정렬·필터링한다. 정렬 기준에 따라 컬럼 구성이 동적으로 변한다 (8개 ↔ 7개).

## 컴포넌트 구조

```
ChartSection                       # 02-탭 스펙에서 정의된 wrapper
└── ChartSectionList                # 본 스펙
    ├── ChartListHeader
    └── ChartListRow × N            # N = 시장 필터 적용 후 항목 수 (최대 100)
        └── Logo                    # 01-지표와 공유
```

## 컴포넌트 명세

### ChartSectionList

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| stocks | Stock[] | O | 원천 데이터 (`useStocks()` 결과) |
| filter | ChartFilterState | O | 시장/정렬/기간 |

**동작:**
1. `applyChartFilter(stocks, filter)`로 정렬·필터를 적용.
2. `<ChartListHeader sort={filter.sort} />` 렌더.
3. 결과 배열을 인덱스+1을 `rank`로 부여하여 `<ChartListRow>` 100개(또는 시장 필터 적용 후 수)로 렌더.
4. 결과 0건이면 별도 안내 없이 빈 영역으로 노출.

### 정렬·필터 헬퍼 (`src/utils/stocks.ts`)

```ts
export function applyChartFilter(stocks: Stock[], filter: ChartFilterState): Stock[];
```

**동작:**
- `filter.market`이 `'국내'` 또는 `'해외'`이면 해당 시장만 남기고, `'전체'`이면 필터링 없음.
- 다음 매핑으로 `metricsByPeriod[filter.period]` 기준 정렬:

| sort | 비교 키 | 방향 |
|------|---------|------|
| 토스증권 거래대금 | `tossTradingValue` | desc |
| 토스증권 거래량 | `tossTradingVolume` | desc |
| 거래대금 | `totalTradingValue` | desc |
| 거래량 | `totalTradingVolume` | desc |
| 급상승 | `changeRate` | desc |
| 급하락 | `changeRate` | asc |

- 결과 0건이면 빈 배열 반환.

### ChartListHeader

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| sort | SortKey | O | 컬럼 구성 결정용 |

**컬럼 구성:**

거래대금/거래량 계열 (`토스증권 거래대금` / `토스증권 거래량` / `거래대금` / `거래량`) — **8개 컬럼**:

| # | 컬럼 헤더 |
|---|-----------|
| 1 | 순위 |
| 2 | 로고 |
| 3 | 종목명 |
| 4 | 현재가 |
| 5 | 등락률 |
| 6 | `<sort> 순` (예: `토스증권 거래대금 순`) |
| 7 | 토스증권 거래 비율 |
| 8 | 토스증권 AI 요약 |

등락률 계열 (`급상승` / `급하락`) — **7개 컬럼**:

| # | 컬럼 헤더 |
|---|-----------|
| 1 | 순위 |
| 2 | 로고 |
| 3 | 종목명 |
| 4 | 현재가 |
| 5 | 등락률 순 |
| 6 | 토스증권 거래 비율 |
| 7 | 토스증권 AI 요약 |

### ChartListRow

| 속성 | 타입 | 필수 | 설명 |
|------|------|------|------|
| rank | number | O | 1부터 시작 |
| stock | Stock | O | 종목 데이터 |
| period | Period | O | 등락률·동적 metric 추출 |
| sort | SortKey | O | 컬럼 구성 결정 |

**셀별 동작:**

| 위치 | 표시 값 |
|------|---------|
| 순위 | `rank` |
| 로고 | `<Logo logoSeed={stock.logoSeed} />` ([01-지표.md](./01-지표.md) 정의) |
| 종목명 | `stock.name` |
| 현재가 | `stock.currentPrice`. 천 단위 콤마 포맷 + **`₩` 접두**. 시장 구분 없이 항상 `₩` 사용 (예: `₩78,900`, `₩245.32`) |
| 등락률 / 등락률 순 | `metricsByPeriod[period].changeRate` (%). **양수에는 `+` 부호 prepend** (예: `+1.23%`), 음수는 부호 그대로 (예: `-1.23%`), 0%는 `0.00%`. 색상은 양수 빨강(`#f04452`)·음수 파랑(`#3182f6`)·**0%는 기본 텍스트 색상** |
| 6번째 컬럼 (거래대금/거래량 계열일 때만) | `sort`에 매핑된 metric 값 (`metricsByPeriod[period].{tossTradingValue \| tossTradingVolume \| totalTradingValue \| totalTradingVolume}`). **천 단위 콤마** 포맷 |
| 토스증권 거래 비율 | `stock.tossSharePercent` (%) |
| 토스증권 AI 요약 | `stock.aiSummary` (한 줄, 넘치면 말줄임) |

**행 동작:**
- 행 클릭 시 `useNavigate('/stocks/{stock.stockId}')`.
- `/stocks/:stockId` 라우트는 [01-지표.md](./01-지표.md)에서 등록한 placeholder를 공유.

## 상태 관리

### 서버 상태

- 본 컴포넌트는 `useStocks()`를 직접 호출하지 않는다.
- 흐름: HomePage(`useStocks()` 1회 호출) → ChartSection → ChartSectionList. 자세한 호출 위치는 [01-지표.md](./01-지표.md) 참조.

### 클라이언트 상태

- 자체 상태 없음.
- 필터 상태는 [02-탭.md](./02-탭.md)의 `ChartSection` 부모 `useState<ChartFilterState>`에서 props로 전달받는다.

### 파생 데이터

- `applyChartFilter(stocks, filter)` 결과를 `useMemo`로 메모이즈 (입력: `stocks`, `filter.market`, `filter.sort`, `filter.period`).
