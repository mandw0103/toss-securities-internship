# 기획서 작성 스킬

## 트리거

민서가 기획서 작성을 요청하거나, Confluence 링크/내용을 전달했을 때 사용한다.

## 입력

민서가 아래 중 하나 이상을 제공한다:

- Confluence 기획서 링크 또는 본문 내용
- 피처명
- 구두 설명

## 작성 절차

1. 민서가 제공한 기획 내용을 정리한다
2. `docs/planning/` 내 기존 파일 번호를 확인하여 다음 번호를 부여한다 (예: `02-home.md`)
3. 아래 **문서 구조**에 맞춰 기획서를 작성한다
4. 작성 완료 후 민서에게 리뷰를 요청한다

## 문서 구조

```markdown
# [피처명]

> 기획서 원본: [Confluence 링크]
> 작성일: YYYY-MM-DD
> 작성자: 민서

## 개요

이 피처가 왜 필요한지, 어떤 문제를 해결하는지 간단히 설명한다.

## 사용자 시나리오

1. 사용자가 ~한다
2. 화면에 ~가 표시된다
3. 사용자가 ~를 클릭하면 ~한다

## 화면 구성

### 화면 1: [화면명]
- 구성 요소 나열
- 각 요소의 동작 설명

## 데이터 요구사항

| 항목 | 타입 | 설명 | 비고 |
|------|------|------|------|

## 비즈니스 규칙

- 규칙 1: ...

## 예외 처리

| 상황 | 처리 방식 |
|------|-----------|

## 인터랙션 요약

| 트리거 | 동작 |
|--------|------|

## 변경 이력

| 날짜 | 내용 |
|------|------|
```

## 규칙

- 민서가 제공하지 않은 내용을 임의로 추가하지 않는다
- 모호한 부분은 `[TODO: 민서 확인 필요]`로 표시하고 리뷰 시 질문한다
- 작성일은 오늘 날짜, 작성자는 항상 `민서`로 기입한다
- 파일명은 피처 순서 기준으로 `XX-피처명.md` 형식을 따른다
