# PR 스킬

## 트리거

민서가 PR 생성을 요청했을 때 사용한다.

## 절차

1. `git log`로 현재 브랜치의 커밋 목록을 확인한다
2. 커밋 내용을 분석하여 PR **제목**과 **본문**을 작성한다
3. 민서에게 추천안을 보여주고 **승인을 받는다**
4. 민서가 승인하면 `gh pr create`로 PR을 생성한다

## PR 본문 형식

```markdown
## Summary
- 변경 사항 요약 (1~3줄)

## Commits
| 해시 | 메시지 |
|------|--------|
| [`abc1234`](커밋 링크) | 커밋 메시지 |
| [`def5678`](커밋 링크) | 커밋 메시지 |
```

## 규칙

- PR 제목은 70자 이내로 작성한다
- 본문에 모든 커밋의 해시, 링크, 메시지를 포함한다
- **민서의 승인 없이 PR을 생성하지 않는다**
- 추천안을 보여준 뒤 반드시 승인 여부를 확인한다
