# 구현 스킬

## 트리거

FE 스펙이 `승인됨` 상태가 된 후, 민서가 구현을 요청했을 때 사용한다.

## 전제 조건

- 대상 피처의 FE 스펙(`docs/spec/XX-피처명.md`)이 존재해야 한다
- 스펙 상태가 **승인됨**이어야 한다
- 승인되지 않은 스펙으로는 구현을 시작하지 않고, 민서에게 승인 여부를 확인한다

## 구현 절차

1. 해당 FE 스펙 문서를 읽고 구현 범위를 파악한다
2. 스펙의 **파일 구조** 섹션에 명시된 파일을 기준으로 작업한다
3. 스펙의 **컴포넌트 명세**에 정의된 props, 동작을 그대로 구현한다
4. 스펙의 **API 연동** 섹션에 정의된 인터페이스와 에러 처리를 따른다
5. 스타일은 **https://tossinvest.com** 실제 서비스를 참고하여 클론한다
6. 구현 완료 후 **구현 보고서**를 출력한다
7. 보고서 출력 후 민서에게 리뷰 진행 여부를 확인한다

## 구현 보고서 형식

```markdown
## 구현 보고서

### 구현한 것
- ...

### 임의 판단한 것
- ...

### 못한 것
- ...
```

## 규칙

- CLAUDE.md 핵심 원칙을 따른다
- 스타일(색상, 간격, 폰트, 레이아웃)은 실제 토스증권 사이트를 기준으로 클론한다
